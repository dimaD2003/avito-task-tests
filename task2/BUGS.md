# Обнаруженные баги

## BUG-001: Пустой ID возвращает 404 вместо 400
**Тест кейс:** ТС-105

**Краткое описание:**  
При выполнении GET-запроса к `/api/1/item/` с пустым id сервис возвращает 404 Not Found, хотя в спецификации ожидается 400 Bad Request.

**Шаги воспроизведения:**  
1. Отправить GET-запрос:  
2. Проанализировать код ответа.

**Фактический результат:**  
- HTTP-код: 404 Not Found  
- Тело ответа: стандартная страница 404 (или пустой объект)

**Ожидаемый результат:**  
- HTTP-код: 400 Bad Request  
- Тело ответа: JSON с полями `status` и `result`, содержащими информацию об ошибке валидации.

**Серьёзность:** Средняя (сервис возвращает некорректный код ответа, что может сбивать клиентов API)  

**Окружение:**  
- Сервис: {{baseUrl}}/api/1/item  
- Тестовое окружение: Windows 10, Python 3.10, pytest 9.0.2


## BUG-002:  Получение статистики удалённого объявления
**Тест кейс:** ТС-305

**Тип:** Особенность / Неопределённое поведение API


**Описание:**  
В тест-кейсе TC-305 ожидалось, что после удаления объявления (DELETE) при запросе статистики по этому id (GET `/api/1/statistic/:id`) будет возвращаться **404**.  

На практике текущая реализация API возвращает **200 OK** с данными статистики, даже если объявление удалено.

---

**Шаги для воспроизведения:**

1. Создать объявление методом POST, получить `id`.
2. Удалить объявление по этому `id` через DELETE `/api/2/item/:id`.
3. Сделать GET `/api/1/statistic/:id`.

---

**Фактический результат:**  
- Статистика возвращается с кодом 200, данные доступны, несмотря на удаление объявления.

**Ожидаемый результат (по текущему тест-кейсу):**  
- GET `/api/1/statistic/:id` → 404 (нет данных для удалённого объявления).

---

**Комментарий / рекомендация:**  
- Поведение API соответствует текущей реализации и не может быть однозначно классифицировано как баг.  
- Необходимо уточнить требования: должна ли статистика удаляться вместе с объявлением, или сохранение статистики после удаления — допустимое поведение.

**Приоритет:** Средний (для уточнения требований).

**Примеры ответов API:**  
- DELETE: 200 / 204  
- GET: 200, тело ответа с данными статистики

## BUG-003: Пустой ID для статистики возвращает 404 вместо 400

**Краткое описание:**

**Фактический результат**

- HTTP-код: 404 Not Found  
- Тело ответа: стандартная страница 404 (или пустой объект)

**Ожидаемый  результат**
- HTTP-код: 400 Bad Request  
- Тело ответа: JSON с полями `status` и `result`, содержащими информацию об ошибке валидации.

**Серьёзность:** Средняя (сервис возвращает некорректный код ответа, что может сбивать клиентов API)  

**Окружение:**  
- Сервис: {{BASE_URL}}/api/1/statistic
- Тестовое окружение: Windows 10, Python 3.10, pytest 9.0.2
